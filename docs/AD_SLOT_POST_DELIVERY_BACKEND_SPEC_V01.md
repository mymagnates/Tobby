# Ad Slot Post Delivery Backend Spec (V0.1)

## 1. Scope

Build a backend service to:

1. ingest posts generated by SP services;
2. deliver eligible posts to PM/PO users in ad slot cards;
3. enforce region matching between post target region and user region;
4. support fixed card size metadata for frontend rendering;
5. count impression and click events;
6. reserve sponsored promotion capability behind feature flags.

Out of scope for V0.1:

1. payment settlement and invoice posting;
2. advertiser self-serve UI;
3. conversion attribution beyond click.

## 2. Runtime Components

1. `sp-ingestion`: webhook/polling entry, idempotent upsert.
2. `post-normalizer`: map multi-source SP payload into a canonical post.
3. `eligibility-engine`: status, time, region, and frequency filter.
4. `ad-feed-api`: returns cards for ad slot rendering.
5. `tracking-api`: impression record and click redirect counter.
6. `analytics-job`: hourly/daily aggregation for dashboard and billing.

## 3. Canonical Card Contract

Each feed item must carry:

1. fixed card spec:
   1. `card_width_px`
   2. `card_height_px`
   3. `image_aspect_ratio`
2. type tag display:
   1. `service_type`
   2. `service_type_label`
   3. `label_position` fixed as `bottom_left`
3. click target:
   1. backend click URL
   2. resolved SP handout URL

Recommended default card spec:

1. `320 x 180` for standard placement;
2. image ratio `16:9`;
3. backend-generated thumbnail to prevent oversized source image impact.

## 4. PostgreSQL DDL (V0.1)

```sql
-- 4.1 SP post source registry
CREATE TABLE sp_sources (
  source_id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source_code         TEXT NOT NULL UNIQUE, -- e.g. "sp_plumbing_v1"
  source_name         TEXT NOT NULL,
  source_type         TEXT NOT NULL CHECK (source_type IN ('webhook','polling')),
  status              TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active','paused','disabled')),
  created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at          TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 4.2 Canonical posts
CREATE TABLE ad_posts (
  post_id                   UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source_id                 UUID NOT NULL REFERENCES sp_sources(source_id),
  source_post_id            TEXT NOT NULL,
  service_type              TEXT NOT NULL, -- plumbing/electrical/hvac/...
  service_type_label        TEXT NOT NULL,
  title                     TEXT NOT NULL,
  summary                   TEXT,
  image_url                 TEXT,
  thumbnail_url             TEXT,
  card_width_px             INTEGER NOT NULL DEFAULT 320,
  card_height_px            INTEGER NOT NULL DEFAULT 180,
  image_aspect_ratio        TEXT NOT NULL DEFAULT '16:9',
  label_position            TEXT NOT NULL DEFAULT 'bottom_left',
  landing_url_template      TEXT NOT NULL,
  handout_id                TEXT,
  source_payload            JSONB NOT NULL DEFAULT '{}'::jsonb,
  status                    TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('draft','active','paused','archived')),
  moderation_status         TEXT NOT NULL DEFAULT 'approved' CHECK (moderation_status IN ('pending','approved','rejected')),
  starts_at                 TIMESTAMPTZ,
  ends_at                   TIMESTAMPTZ,
  priority                  INTEGER NOT NULL DEFAULT 0,
  quality_score             NUMERIC(5,4) NOT NULL DEFAULT 0.5000,
  delivery_type             TEXT NOT NULL DEFAULT 'organic' CHECK (delivery_type IN ('organic','sponsored')),
  campaign_id               UUID NULL, -- reserved for sponsored
  created_at                TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at                TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (source_id, source_post_id)
);

CREATE INDEX idx_ad_posts_active_window
  ON ad_posts(status, moderation_status, starts_at, ends_at);

CREATE INDEX idx_ad_posts_delivery_type
  ON ad_posts(delivery_type, campaign_id);

-- 4.3 Region targeting
CREATE TABLE ad_post_target_regions (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  post_id             UUID NOT NULL REFERENCES ad_posts(post_id) ON DELETE CASCADE,
  region_level        TEXT NOT NULL CHECK (region_level IN ('country','state','city')),
  region_code         TEXT NOT NULL,  -- e.g. US, US-TX, US-TX-AUS
  created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (post_id, region_level, region_code)
);

CREATE INDEX idx_ad_post_target_regions_lookup
  ON ad_post_target_regions(region_level, region_code, post_id);

-- 4.4 User region snapshot for matching
CREATE TABLE user_region_profiles (
  user_id             TEXT PRIMARY KEY,
  country_code        TEXT NOT NULL,
  state_code          TEXT,
  city_code           TEXT,
  timezone            TEXT,
  updated_at          TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 4.5 Slot config and quota
CREATE TABLE ad_slot_configs (
  slot_id                     TEXT PRIMARY KEY, -- e.g. "pm_feed_top"
  status                      TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active','paused')),
  card_width_px               INTEGER NOT NULL DEFAULT 320,
  card_height_px              INTEGER NOT NULL DEFAULT 180,
  image_aspect_ratio          TEXT NOT NULL DEFAULT '16:9',
  max_items                   INTEGER NOT NULL DEFAULT 1,
  sponsored_ratio_numerator   INTEGER NOT NULL DEFAULT 1,
  sponsored_ratio_denominator INTEGER NOT NULL DEFAULT 5,
  frequency_cap_window_hours  INTEGER NOT NULL DEFAULT 24,
  frequency_cap_per_post      INTEGER NOT NULL DEFAULT 1,
  fallback_post_id            UUID NULL REFERENCES ad_posts(post_id),
  updated_at                  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 4.6 Feed delivery logs (impression token issuance)
CREATE TABLE ad_delivery_logs (
  delivery_id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  request_id            TEXT NOT NULL,
  slot_id               TEXT NOT NULL REFERENCES ad_slot_configs(slot_id),
  user_id               TEXT NOT NULL,
  post_id               UUID NOT NULL REFERENCES ad_posts(post_id),
  delivery_type         TEXT NOT NULL CHECK (delivery_type IN ('organic','sponsored')),
  campaign_id           UUID NULL,
  region_match_level    TEXT NOT NULL CHECK (region_match_level IN ('country','state','city')),
  rank_score            NUMERIC(7,5) NOT NULL DEFAULT 0,
  impression_token      TEXT NOT NULL UNIQUE,
  served_at             TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_ad_delivery_logs_user_slot_time
  ON ad_delivery_logs(user_id, slot_id, served_at DESC);

-- 4.7 Impression events
CREATE TABLE ad_impression_events (
  impression_id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  impression_token      TEXT NOT NULL UNIQUE,
  delivery_id           UUID NOT NULL REFERENCES ad_delivery_logs(delivery_id),
  user_id               TEXT NOT NULL,
  post_id               UUID NOT NULL REFERENCES ad_posts(post_id),
  slot_id               TEXT NOT NULL,
  event_time            TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  viewport_ratio        NUMERIC(4,3),
  dwell_ms              INTEGER,
  user_agent            TEXT,
  ip_hash               TEXT
);

CREATE INDEX idx_ad_impression_post_time
  ON ad_impression_events(post_id, event_time DESC);

-- 4.8 Click events
CREATE TABLE ad_click_events (
  click_id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  impression_token      TEXT NOT NULL,
  delivery_id           UUID NOT NULL REFERENCES ad_delivery_logs(delivery_id),
  user_id               TEXT NOT NULL,
  post_id               UUID NOT NULL REFERENCES ad_posts(post_id),
  slot_id               TEXT NOT NULL,
  event_time            TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  referer               TEXT,
  user_agent            TEXT,
  ip_hash               TEXT
);

CREATE UNIQUE INDEX uq_ad_click_once_per_impression
  ON ad_click_events(impression_token);

-- 4.9 Sponsored campaign reserve (feature-flagged)
CREATE TABLE promo_campaigns (
  campaign_id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  advertiser_sp_id      TEXT NOT NULL,
  campaign_name         TEXT NOT NULL,
  status                TEXT NOT NULL DEFAULT 'draft' CHECK (status IN ('draft','active','paused','ended')),
  objective             TEXT NOT NULL DEFAULT 'traffic' CHECK (objective IN ('traffic','clicks')),
  budget_total_cents    BIGINT,
  budget_daily_cents    BIGINT,
  bid_mode              TEXT NOT NULL DEFAULT 'cpc' CHECK (bid_mode IN ('cpc','cpm')),
  bid_amount_cents      BIGINT,
  starts_at             TIMESTAMPTZ,
  ends_at               TIMESTAMPTZ,
  created_at            TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at            TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_promo_campaigns_status_window
  ON promo_campaigns(status, starts_at, ends_at);
```

## 5. Matching and Delivery Rules

1. hard filters:
   1. `ad_posts.status = active`
   2. `moderation_status = approved`
   3. current time in `[starts_at, ends_at]` (null means open bound)
2. region matching:
   1. user region and post target region must match at configured level;
   2. default strategy: `city > state > country` fallback;
   3. record match level into `ad_delivery_logs.region_match_level`.
3. frequency cap:
   1. same `user_id + post_id + slot_id` not served more than configured limit in cap window.
4. sponsored reservation:
   1. if feature flag off, only `organic`;
   2. if on, apply slot ratio (for example 1 of 5).

## 6. API Contract (V0.1)

Base path: `/api`

### 6.1 Ingest SP Post

`POST /sp/posts/ingest`

Purpose:

1. idempotent ingest/upsert canonical post from SP payload.

Request:

```json
{
  "source_code": "sp_plumbing_v1",
  "source_post_id": "p_98421",
  "service_type": "plumbing",
  "service_type_label": "Plumbing",
  "title": "24h Leak Fix",
  "summary": "Licensed plumber available tonight.",
  "image_url": "https://cdn.example.com/post/p_98421.jpg",
  "landing_url_template": "https://sp.example.com/handout/h123?post_id={post_id}",
  "handout_id": "h123",
  "starts_at": "2026-03-22T10:00:00Z",
  "ends_at": "2026-03-29T10:00:00Z",
  "target_regions": [
    { "region_level": "state", "region_code": "US-TX" },
    { "region_level": "city", "region_code": "US-TX-AUS" }
  ],
  "priority": 20,
  "delivery_type": "organic"
}
```

Response:

```json
{
  "request_id": "req_01",
  "post_id": "1d2f4c8a-3d5a-4e8a-9c95-4e0bc6a0e6a1",
  "upserted": true
}
```

### 6.2 Fetch Ad Slot Feed

`GET /ad-slots/{slot_id}/feed?limit=1`

Headers:

1. `X-User-Id`
2. `X-User-Role` (`pm_po`)

Response:

```json
{
  "request_id": "req_02",
  "slot_id": "pm_feed_top",
  "items": [
    {
      "delivery_id": "8d84d3a7-f9d7-4a72-8a74-57918e64da4d",
      "impression_token": "itk_6fca3f6f4c6a44c5",
      "post_id": "1d2f4c8a-3d5a-4e8a-9c95-4e0bc6a0e6a1",
      "delivery_type": "organic",
      "campaign_id": null,
      "title": "24h Leak Fix",
      "summary": "Licensed plumber available tonight.",
      "image_url": "https://cdn.example.com/post/p_98421_thumb.jpg",
      "service_type": "plumbing",
      "service_type_label": "Plumbing",
      "label_position": "bottom_left",
      "card_spec": {
        "width_px": 320,
        "height_px": 180,
        "image_aspect_ratio": "16:9"
      },
      "tracking": {
        "impression_url": "/api/ad-events/impression",
        "click_url": "/api/ad-events/click/itk_6fca3f6f4c6a44c5"
      }
    }
  ]
}
```

### 6.3 Record Impression

`POST /ad-events/impression`

Request:

```json
{
  "impression_token": "itk_6fca3f6f4c6a44c5",
  "viewport_ratio": 0.75,
  "dwell_ms": 1200
}
```

Behavior:

1. idempotent by `impression_token`;
2. first write returns `recorded=true`, duplicate returns `recorded=false`.

Response:

```json
{
  "request_id": "req_03",
  "recorded": true
}
```

### 6.4 Click Redirect + Count

`GET /ad-events/click/{impression_token}`

Behavior:

1. create click event once per impression token;
2. increment click counter;
3. resolve handout URL and return `302 Location: <sp handout url>`.

Fallback:

1. if token invalid, return `404 INVALID_IMPRESSION_TOKEN`.

### 6.5 Internal Sponsored Campaign (reserved)

`POST /internal/promo-campaigns`

Feature flag required:

1. `ads.sponsored.enabled=true`.

Request sample:

```json
{
  "advertiser_sp_id": "u-sp-7",
  "campaign_name": "Spring HVAC Promo",
  "objective": "clicks",
  "budget_daily_cents": 5000,
  "bid_mode": "cpc",
  "bid_amount_cents": 200,
  "starts_at": "2026-04-01T00:00:00Z",
  "ends_at": "2026-04-30T23:59:59Z"
}
```

## 7. Counting and Reporting

Primary counters:

1. served count from `ad_delivery_logs`;
2. impression count from `ad_impression_events`;
3. click count from `ad_click_events`;
4. CTR = clicks / impressions.

Breakdown dimensions:

1. `slot_id`
2. `service_type`
3. `region_code`
4. `delivery_type`
5. `campaign_id` (nullable for organic)
6. hourly and daily time grain.

## 8. Safety and Compliance

1. `moderation_status` must be `approved` before serving.
2. sponsored items must expose promotion badge in frontend (`delivery_type=sponsored`).
3. keep immutable event logs for reconciliation.
4. do not accept raw IP storage; store hashed or tokenized value only.

## 9. Rollout Plan

1. phase 1:
   1. organic ingest, region matching, feed, impression, click redirect.
2. phase 2:
   1. sponsored fields enabled in schema;
   2. sponsored API internal only.
3. phase 3:
   1. billing settlement and advertiser self-serve.

## 10. Open Configuration Defaults

1. default slot: `pm_feed_top`;
2. default match level: city fallback to state;
3. frequency cap: `1` per post per user per `24h`;
4. sponsored ratio default: `1/5` (only when enabled);
5. click dedupe window: one click per impression token.
